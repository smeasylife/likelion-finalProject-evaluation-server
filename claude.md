# 멋사 13기 프로젝트 최종 발표 채점 시스템 - 백엔드 명세서
     ## 1. 시스템 개요
     본 백엔드 시스템은 멋사 13기 4개 팀의 최종 발표 채점을 위한 API 서비스입니다. 심사위원과 아기사자들이 각 팀을 15개 항목(디자인 5개, 개발 7개, 공통 3개)에 대해 1-5점으로 평가하며,
     평가 결과를 집계하여 순위를 산출합니다.
     ## 2. 기술 스택
     - **언어**: Java 17+
     - **프레임워크**: Spring Boot 3.x
 

  주요 내용:

  📋 데이터베이스 설계

  - 2개 테이블: teams, evaluations (evaluations가 다대일로 teams를 가지게 된다. evaluations는 평가자의 역할(string), 이름, 개발분야 총 점수, 디자인분야 총 점수, 공통분야 총 점수를 컬럼으로 가진다 )

  🔌 API 엔드포인트

  - POST /api/evaluations - 프론트엔드 handleSubmit 함수와 연동 (사용자가 팀에 대해 평가한 데이터를 보내서 db에 반영한다)
  - GET /api/results - ResultPage.jsx의 결과 조회와 연동 (team 기준으로 evaluations를 불러와서 총 점수를 계산해서 return 한다. 모든 팀의 결과를 보여준다.)


  ⚙️ 핵심 기능

  - 중복 제출 방지 (같은 평가자가 같은 팀 재평가 불가)
  - 카테고리별 상세 점수 집계
  - 동시성을 생각한 점수 반영 로직
  - 완벽한 예외 처리
